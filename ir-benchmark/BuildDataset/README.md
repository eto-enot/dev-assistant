## Утилита для построения набора данных для бенчмарка

Для тестирования алгоритма разбиения файлов исходного кода на сегменты с учетом семантики используется набор данных, подготовленных по нескольким (произвольным) репозиториям. Утилита обрабатывает репозиторий (каталог с исходным кодом), удаляя из всех файлов все docstring. Затем составляет набор данных в формате xml, содержащий:
- `file`: имя исходного файла.
- `member`: имя типа данных, функции, метода и т.п.
- `type`: `M` - функция, метод, оператор и т.п., `T` - тип данных.
- `summary`: исходный docstring.

Исходный docstring используется в бенчмарке как запрос для ретривера. Задача ретривера - найти тип данных (метод, функцию и т.п.), соответствующий описанию из docstring.
